# 交易复盘工作流

本工作流用于处理用户上传的 TradingView 截图和交易叙述，完成"数据驱动"的复盘。项目仓库：`https://github.com/phoe8lin/tradereview`，本地路径 `/Volumes/external/trade/交易复盘/`。

## 两种模式

根据用户输入判断应走哪条路径：

| 用户提供 | 模式 | 编号规则 |
|---|---|---|
| TV 截图 + 单笔交易（明确 entry/stop/take 和锚定 K 线） | **单笔模式**（下述主流程） | 锚 K = `A0`，前负后正 |
| "帮我复盘今天" / 扫描日内多笔机会 / 无截图只说日期 | **日级模式**（见文末"日级复盘流程"） | 全日 `B00-B95`（15m），辅周期 `E00-EN` |

两种模式产物都落到同一个 `reviews/<date>/` 目录，互不冲突。

---

## 单笔模式核心原则

1. **锚 K 为 A0，前负后正**：`A0 = 用户截图中的"位置1 / 锚定 K 线"`；之前是 `A-1, A-2, ...`，之后是 `A1, A2, ...`。所有分析必须以编号引用 K 线，**禁止用自然语言描述位置**（如"左边那根""中间那根"），避免认错。
2. **数据先行**：任何分析结论都必须能在 `reviews/<日期>/data/<trade_id>.parquet` 中找到对应的列值。**禁止凭直觉/截图肉眼估算**。
3. **统一时区 UTC+8**。
4. **默认交易所 Binance USDT-M 永续**（如截图明确是其他交易所/现货则调整）。

## 步骤

### 步骤 1：从用户输入中提取关键信息

需要从截图 + 文字中确定：

- 标的（`base/quote`，如 BTC/USDT）
- 周期（如 5m / 1h）
- 交易所（默认 binance，永续/现货）
- **锚定 K 线时间**（UTC+8 的精确时间，**必须精确到分钟**）
- 方向（long / short）
- 入场价 / 止损价 / 止盈价

**如果锚定时间不精确，必须先向用户确认**，不要猜。价格数据不对会导致后续所有分析失效。

### 步骤 1.5：保存截图到 raw/ 目录（可选但强烈建议）

**Cascade 无法代你保存图片**，需要你自己操作：

1. 在本地把 TV 截图另存为 `reviews/<date>/raw/<trade_id>.png`（或 .jpg/.jpeg/.webp）
   - 例：`reviews/2026-04-19/raw/HYPE_5m_001.png`
2. `review_builder` 会自动检测该文件并在 `review.md` 中嵌入 Markdown 图片引用
3. 之后在 IDE 或 GitHub 里打开 `review.md` 就能直接看到截图

如果步骤 1 跳过，review.md 就没有截图区（其他内容不受影响）。

### 步骤 2：生成复盘包

```bash
# 在项目根目录执行
/opt/anaconda3/envs/trade/bin/python -m tools.review_builder \
  --date YYYY-MM-DD \
  --trade-id <标的>_<周期>_<序号> \
  --base <BASE> --quote USDT \
  --timeframe <周期> \
  --anchor "YYYY-MM-DD HH:MM" \
  --entry <价> --stop <价> --take <价> \
  --direction <long|short> \
  --notes "<一句话概述>"
```

产物：
- `reviews/<date>/data/<trade_id>.parquet` — 带指标和特征的完整 K 线
- `reviews/<date>/replicated/<trade_id>.html` — Plotly 交互图
- `reviews/<date>/trades/<trade_id>.yaml` — 结构化元数据
- `reviews/<date>/review.md` — 当日复盘累计文档（自动 append 一段）

### 步骤 3：数据核验（防认错）

生成后**立即**用 pandas 读 parquet 打印锚 K 前后 10 根数据，核对：

- 锚 K 的 OHLC 是否与截图显示的 hover 值一致（误差 < 0.1%）
- 如果不一致 → 锚定时间错了，回步骤 1 重新确认

```bash
/opt/anaconda3/envs/trade/bin/python -m tools.calibration.stochrsi_check \
  --data reviews/<date>/data/<trade_id>.parquet --window 10
```

### 步骤 4：分析

**引用规则**：
- 所有 K 线引用使用编号（A-3, A0, A5 等），**且必须附带时间点**避免用户在 TV 图上多标记时搞混
  - 同日内：`A0 (14:50)`、`A-2 (14:40)`
  - 跨日：`A0 (04-20 14:50)`
  - 表格中若已有"时间"列可省略 inline 时间，正文引用必须补
- 每个结论后括号注明依据字段，如"A0 (14:50) Wave 46.86 严重超买" 而不是"A0 超买"
- K 线形态特征字段要点：
  - `upper_wick_ratio` = 上影比例，`lower_wick_ratio` = 下影比例（**注意别搞反**）
  - `body_ratio` = 实体比例
  - `vol_vs_ma` = 成交量/20根均量，>1.5 视为放量
  - `engulf` ∈ {`bull`, `bear`, ``} = 吞没形态
  - `wave` ∈ [-50, 50]，>+40 超买，<-40 超卖

### 步骤 5：用户交互讨论

让用户指认位置2/位置3 等关键 K 线的编号后再展开分析，不要自己猜。

### 步骤 6：落实结论到 review.md

分析完成后追加到 `reviews/<date>/review.md`：
- 关键 K 线映射表（位置 ↔ 编号 ↔ 时间 ↔ 关键数据）
- 本单结论（3-5 条）
- **心得标签**（`#xxx #yyy` 格式，方便日后跨复盘检索）

## 常见错误与防御

| 错误 | 防御 |
|---|---|
| 看反 `upper/lower_wick_ratio` | 分析前先在脑海中复述字段含义 |
| 锚时间猜错导致价格区间不匹配 | 步骤 3 强制核验 |
| 用自然语言"左边那根"误指 | 全部用 A 编号 |
| 指标读数凭印象 | 每个数字都从 parquet 取 |

## 日级复盘流程

**触发条件**：用户没给截图，也没给具体交易参数，而是说"复盘一下今天 XX"、"扫一下今天 HYPE 15m"、"今天有哪些机会"、"我想看看今天整天"——走日级模式。

### 步骤 D1：确认基本要素

只需要和用户确认：
- **日期**（UTC+8，默认"今天"）
- **标的**（如 HYPE/USDT）
- **主周期**（默认 15m，可改 5m/1h）
- **辅周期窗口**（可选，如某段时间想看 5m 细节）
- **交易所**（默认 binance futures）

### 步骤 D2：一键生成数据 + 骨架

```bash
/opt/anaconda3/envs/trade/bin/python -m tools.day_review_builder \
  --date YYYY-MM-DD --base <BASE> --timeframe 15m \
  [--supp-tf 5m --supp-window "HH:MM-HH:MM"]
```

产物：
- `reviews/<date>/data/day_<BASE>_<TF>.{parquet,txt}` — 主周期全日 + EMA + Wave + 形态 + 订单流，编号 `B00-B95`
- `reviews/<date>/data/supp_<BASE>_<SUPP_TF>.{parquet,txt}` — 辅周期窗口，编号 `E00-EN`（可选）
- `reviews/<date>/review.skeleton.md` — 每次刷新的骨架，含自动识别的长影/放量/吞没/EMA100 触点/Wave 峰谷
- `reviews/<date>/review.md` — **首次创建复制骨架；已存在不覆盖**

**增量更新**：同一日可多次运行，parquet/skeleton 都会刷新，review.md 的手写内容受保护。

### 步骤 D3：读表、识别、讨论

1. 读 `data/day_<BASE>_<TF>.txt` 掌握全日走势
2. 参考 `review.skeleton.md` 的"二、日内关键位与信号候选"自动识别列表（长影/放量/吞没/EMA100 触点/Wave 峰谷）快速定位**候选机会 K 线**
3. 逐个与用户讨论：
   - 你看到了什么结构？
   - 该位置是否符合你的交易系统（趋势/形态/量能/wave）？
   - 入场/止损/止盈分别放哪？R:R 够不够？
   - 信号强度 A+/A/B/C 自评，对应仓位 1R/1R/0.5R/跳过
4. 后验验证：用 parquet 数据确认止损是否触发、止盈是否达成、持仓时长、实际 delta 印证

### 步骤 D4：沉淀到 review.md

把讨论结果落到 `reviews/<date>/review.md`，建议结构：

1. **日概览**（结构定性：趋势日/震荡日/反转日 + 主方向 + 关键位）
2. **关键位与候选信号**（映射表）
3. **交易点位全景表**（时间 / 方向 / 入场 / 止损 / 止盈 / R:R / 信号强度 / 可交易 / 决策 / 结果）
4. **订单流与形态互证**（背离点、吸收点、推动-衰竭模板案例）
5. **方法论沉淀**（今日最重要的 1-3 条认知）
6. **辅周期补充**（如果用了）
7. **心得标签** `#xxx #yyy`
8. **产物清单**

骨架中已有章节占位符，填入内容即可。`review.skeleton.md` 保留供下次增量对照。

### 日级模式引用规则

- K 线引用用日级编号 `B22 (05:30)` / `E32 (22:40)`，**附带时间**
- 多笔对比时列表格，按时间顺序排列
- 每个交易点独立评价**信号强度**和**可交易性 R:R**，不混为一谈
- 后验验证必须基于 parquet 数据，不凭记忆
